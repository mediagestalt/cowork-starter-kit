# Friday Weekly Review — Task Prompt Template

Paste this into the prompt field when creating your Friday weekly review scheduled task. Replace all `[PLACEHOLDER]` values with your own information before saving.

---

## PROMPT TEXT

OBJECTIVE
Every Friday afternoon, run a weekly review for [YOUR_NAME]: summarize what got done this week, check project status, maintain the projects file, and close out the day with a mood check.

COORDINATION
You are not the only writer of these files — a live planning session and sibling scheduled tasks read and write them too. Before adding any task, note, completion, or chase, check whether it already exists for the relevant date. If it does, do not add a second copy — skip it, or update the existing entry in place. Only add what is genuinely missing.

FILES
- Work log: [YOUR_WORK_LOG_PATH]
- Projects (your view): [YOUR_PROJECTS_PATH]
- Projects (context): [YOUR_PROJECTS_CONTEXT_PATH]

CALENDAR
Calendar system: [YOUR_CALENDAR_NAME]
Calendar ID(s): [YOUR_CALENDAR_ID]

STEP 1 — Read the work log. Scan this week's entries (Monday through today). Collect:
- All tasks marked [x] this week (completed)
- Any tasks still marked [ ] or [/] (open or in progress)
- Any notable entries from "Other notes" sections

STEP 2 — Read +-projects.md. For each active project, note the most recent status entry and the next action.

STEP 3 — Read +-projects-context.md for any deadlines or open threads coming up in the next two weeks.

STEP 4 — Pull next week's calendar events.

STEP 5 — Compose the weekly review message. Sections:

**This week — what got done**
Group completed tasks by project or theme. Warm framing — this is a summary of real progress, not a report card. Lead with what's real.

**Still open**
Any tasks that didn't get closed this week. No judgment — just what's carrying forward. Include unanswered chase tasks; if someone hasn't replied after two chases, suggest an escalation (phone, different contact) rather than a third identical email.

**Project pulse**
One line per active project: where things stand and the next action. Check the Parked section if there is one: flag any parked project whose re-entry trigger has arrived (date passed, person returned). Flag lump deadlines: any task sitting on a deadline date with no work sessions scheduled before it — offer to decompose it.

**Next week**
Key dates from the calendar plus anything that needs to move on the project next actions.

**How's the week landing?**
One question: how did this week feel overall? Ask [YOUR_NAME] to reply with a word or sentence — and that you'll log it.

STEP 6 — When [YOUR_NAME] replies:
- Append a dated status entry to any project they give an update on in the projects file — under that project's Status block, append, don't overwrite. Keep the last 2–3 entries per project; prune older ones.
- Run the weekly dedupe sweep: scan this week's tasks and Other notes across all day headings for entries that say the same thing and merge them; flag anything ambiguous rather than deleting.
- Add a Friday Evening entry to today's work log under "Other notes": **Mood:** [their word or phrase]

CONSTRAINTS
- Read only (files) until [YOUR_NAME] replies. Update the work log only after they reply with their end-of-week mood.
- Read the live work log only. Do not read files in archive/ — this review covers the current week.
- Don't ask multiple questions. One exchange: send the review, ask one question, log the answer.
- No preamble. Start with the review content.
