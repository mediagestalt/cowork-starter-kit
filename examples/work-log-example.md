# Work Log 2026

Daily task tracking. Obsidian Tasks plugin format.

Task syntax:
- [ ] Task description [due:: YYYY-MM-DD]
- [x] Completed task [completion:: YYYY-MM-DD]
- [/] Task in progress
- [-] ❌ YYYY-MM-DD  (cancelled)

Priority: ⏫ high  🔼 medium  (no marker = normal)

Conventions: capture tasks the moment they come up — a new task goes in as a checkbox with a due date right away, even weeks ahead; the tasks dashboard (+-Tasks.md) pulls every open task into one view regardless of which day's entry it sits in. Meetings go in as time-stamped blocks with narrative notes underneath (action items from a meeting are new tasks, so they get checkboxes). The one thing that never gets a checkbox: work already done that was never a task — emails sent, informal conversations, things done on the fly go as plain narrative bullets under "Other notes."
